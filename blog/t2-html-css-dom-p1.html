An Analogy

An analogy for HTML, CSS and DOM could be a book. The HTML would be the content of the book, the actual story. It drives the narrative of the webpage like the words in the book.

CSS is what the book actually looks like. The margins in the page, the front cover, how big the words are.

The DOM is the Document Object Model. In terms of the book analogy, it is the actual physical copy of the book, the one that exists in the real world and the one that we as 'readers' are ultimately interacting with.

Boxifying

Boxifying a design is where you separate all the components of a design into separate and distinct boxes. Boxes can contain other boxes, and there may be boxes inside of boxes inside of boxes!

The tree hierarchical structure of an HTML webpage lends itself to the box model, where all the different elements can be thought of as boxes. This helps in turning a design mockup into real HTML!
